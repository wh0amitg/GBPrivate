GoreBox Private Server

A beginner's guide to setting up your own private GoreBox server — covering tools, version downloading, coding, and backend configuration.

BepInEx IL2CPP MinHook / HarmonyLib PlayFab Photon PUN2 C++ / C# .NET 4.7.2

Chapters

Chapter 01

Required Software

Before you can do anything, you need three tools installed and ready. Don't skip any of them — each handles a different part of the process.

1.
dnSpy

Used to inspect Assembly-CSharp.dll — the game's compiled code. This lets you find method names like VersionValidator that you'll hook. Download from the official GitHub page.

2.
Visual Studio 2022

Used to write and compile your BepInEx plugin as a .dll. Do NOT confuse this with Visual Studio Code — they are completely different programs.

3.
BepInEx IL2CPP (v6+)

The mod loader that injects code into the game. GoreBox runs on IL2CPP, so you must use an IL2CPP build — the standard GitHub release won't work.

⚠ Important Download BepInEx from builds.bepinex.dev. Always pick the IL2CPP build. Any version ≥ 6 works.

Chapter 02

Downloading a Specific Version via SteamDB

Your private server needs to run on a specific GoreBox version. If you own the game on Steam, you can download any old depot directly through the Steam console.

1.
Open the Steam Console

Press Win + R and enter steam://open/console — or paste that into your browser's address bar.

2.
Find the Manifest on SteamDB

Go to the GoreBox SteamDB page and find the depot for the version you want. For example, V15.15 has manifest 3348281417896630206.

3.
Run this in the Steam console

Replace the manifest ID with the one you found for your target version.

Steam Console
download_depot 2027330 2027331 3348281417896630206
What these numbers mean 2027330 = GoreBox App ID  ·  2027331 = Depot ID  ·  last number = Manifest ID for your chosen version

Chapter 03

Coding the Plugin

You'll write a BepInEx plugin that replaces GoreBox's default Photon and PlayFab credentials with your own.

1.
Install BepInEx and generate interop files

Drop BepInEx into your GoreBox folder, launch the game once, wait for it to generate interop files, then close.

2.
Create a Class Library project in Visual Studio

Target .NET 4.7.2. Name it whatever. Set the build config to Release, not Debug.

3.
Find VersionValidator in dnSpy

Open Assembly-CSharp.dll from GoreBox_Data/Managed. Search for VersionValidator and note the lifecycle hook — it varies by version.

Hook names vary by version Older versions may use Start or Awake; newer ones often use OnEnable or OnDisable. Always check in dnSpy first.

Below is the full plugin. Replace YOUR_METHOD_NAME_HERE with what you found in dnSpy, and fill in your Photon / PlayFab credentials.

C# — Main Plugin
using BepInEx;
using BepInEx.Unity.IL2CPP;
using HarmonyLib;
using Photon.Pun;
using PlayFab;
using PlayFab.ClientModels;
using UnityEngine.Networking;

namespace GoreBoxMain
{
    [BepInPlugin("GoreBox", "GoreBoxMain", "1.0")]
    public class GoreBoxClass : BasePlugin
    {
        public override void Load()
        {
            Harmony harmony = new Harmony("Harmony");
            harmony.PatchAll();
        }

        [HarmonyPatch(typeof(VersionValidator), "YOUR_METHOD_NAME_HERE")]
        public static class VersionValPatch
        {
            private static bool Prefix(VersionValidator __instance)
            {
                // Replace Photon credentials with your own
                PhotonNetwork.PhotonServerSettings.AppSettings.AppIdRealtime = "YOUR_PHOTON_APP_ID";
                PhotonNetwork.PhotonServerSettings.AppSettings.AppIdVoice = "YOUR_PHOTON_VOICE_ID";

                // Replace PlayFab credentials with your own
                PlayFabSettings.PlayFabSharedPrivate.TitleId = "YOUR_TITLE_ID";
                PlayFabSettings.PlayFabSharedPrivate.ProductionEnvironmentUrl = "https://YOUR_TITLE_ID.playfabapi.com";

                LoginWithCustomIDRequest loginRequest = new LoginWithCustomIDRequest
                {
                    CreateAccount = new Il2CppSystem.Nullable<bool>(true),
                    CustomId = UnityEngine.SystemInfo.deviceUniqueIdentifier
                };

                System.Action<LoginResult> onSuccess = delegate (LoginResult result)
                {
                    __instance.LoginResult(result); // Find correct LoginResult in dnSpy
                };
                System.Action<PlayFabError> onError = delegate (PlayFabError err)
                {
                    __instance.PlayFabError(err); // Find correct PlayFabError in dnSpy
                };

                PlayFabClientAPI.LoginWithCustomID(loginRequest, onSuccess, onError, null, null);

                return false; // Skip the original method
            }
        }
    }
}
⚠ LoginResult / PlayFabError names differ per version If multiplayer doesn't work, look for other LoginResult or PlayFabError methods inside VersionValidator in dnSpy. They get renamed in almost every update.

void SetupProxy() {
    char sysPath[MAX_PATH];
    GetSystemDirectoryA(sysPath, MAX_PATH);
    HMODULE hReal = LoadLibraryA((std::string(sysPath) + "\\version.dll").c_str());
    if (!hReal) return;

    o_GetFileVersionInfoA    = (decltype(o_GetFileVersionInfoA))   GetProcAddress(hReal, "GetFileVersionInfoA");
    o_GetFileVersionInfoByHandle = (decltype(o_GetFileVersionInfoByHandle))GetProcAddress(hReal, "GetFileVersionInfoByHandle");
    o_GetFileVersionInfoExA  = (decltype(o_GetFileVersionInfoExA)) GetProcAddress(hReal, "GetFileVersionInfoExA");

uintptr_t GameAssembly = 0;
uintptr_t offset_ConnectUsingSettings = 0x0; // Photon.Pun.PhotonNetwork$$ConnectUsingSettings — get from IL2CPPDumper

struct Il2CppString { void* k; void* m; int32_t len; wchar_t chars[1]; };

using il2cpp_string_new_t = Il2CppString* (*)(const char*);
il2cpp_string_new_t il2cpp_string_new = nullptr;

template <typename T>
void WriteMem(uintptr_t addr, T val) {
    DWORD old;
    VirtualProtect((LPVOID)addr, sizeof(T), PAGE_EXECUTE_READWRITE, &old);
    *(T*)addr = val;
    VirtualProtect((LPVOID)addr, sizeof(T), old, &old);
}

using o_photon_hook_t = void(*)(void*, bool, void*);
o_photon_hook_t o_photon_hook = nullptr;

void photon_hook(void* _appSettings, bool startInOfflineMode, void* mi) {
    if (_appSettings && il2cpp_string_new != nullptr) {
        WriteMem((uintptr_t)_appSettings + 0x10, il2cpp_string_new("")); // PUN2 App ID — put yours here
        WriteMem((uintptr_t)_appSettings + 0x28, il2cpp_string_new("")); // Voice App ID — put yours here
    }
    return o_photon_hook(_appSettings, startInOfflineMode, mi);
}

void Main() {
    while (!(GameAssembly = (uintptr_t)GetModuleHandleA("GameAssembly.dll"))) {
        Sleep(200);
    }

    HMODULE hGA = (HMODULE)GameAssembly;
    il2cpp_string_new = (il2cpp_string_new_t)GetProcAddress(hGA, "il2cpp_string_new");

    MH_Initialize();

    uintptr_t targetAddr = GameAssembly + offset_ConnectUsingSettings;
    if (MH_CreateHook((LPVOID)targetAddr, (LPVOID)&photon_hook, (LPVOID*)&o_photon_hook) == MH_OK) {
        MH_EnableHook((LPVOID)targetAddr);
    }
}

BOOL APIENTRY DllMain(HMODULE hMod, DWORD reason, LPVOID lpRes) {
    if (reason == DLL_PROCESS_ATTACH) {
        DisableThreadLibraryCalls(hMod);
        SetupProxy();
        CreateThread(0, 0, (LPTHREAD_START_ROUTINE)Main, 0, 0, 0);
    }
    return TRUE;
}
⚠ Fill in the offsets! The offset_ConnectUsingSettings value must be obtained from IL2CPPDumper on your specific V13 build. Leave it as 0x0 and the hook will crash.

template <typename T>
void WriteMem(uintptr_t addr, T val) {
    DWORD old;
    VirtualProtect((LPVOID)addr, sizeof(T), PAGE_EXECUTE_READWRITE, &old);
    *(T*)addr = val;
    VirtualProtect((LPVOID)addr, sizeof(T), old, &old);
}
