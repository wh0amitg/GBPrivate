Chapter 01

Required Software

Before you can do anything, you need three tools installed and ready. Don't skip any of them — each one handles a different part of the process.

1.
dnSpy

Used to inspect Assembly-CSharp.dll — the game's compiled code. This is how you find method names like VersionValidator that you'll need to hook. Download it from the official GitHub page.

2.
Visual Studio 2022

Used to write and compile your BepInEx plugin as a .dll file. Do not confuse this with Visual Studio Code — they are completely different programs. You need the full Visual Studio IDE.

3.
BepInEx (IL2CPP build, version 6+)

The mod loader that lets you inject code into the game. GoreBox runs on IL2CPP, so you must use an IL2CPP build — the standard GitHub release will not work.

⚠ Important Download BepInEx from the official bleeding-edge build server at builds.bepinex.dev. Always pick the IL2CPP build. Any version ≥ 6 works.

Chapter 02

Downloading a Specific Version via SteamDB

Your private server needs to run on a specific GoreBox version. If you own the game on Steam, you can download any old depot directly through the Steam console.

1.
Open the Steam Console

Press Win + R and enter steam://open/console — or paste that into your browser's address bar. Either way works.

2.
Find the Manifest on SteamDB

Go to the GoreBox SteamDB page and find the depot entry for the version you want. For example, V15.15 has the manifest ID 3348281417896630206.

3.
Download via Steam Console

In the Steam console, run this command (replace the manifest ID with the one you found):

Steam Console 
download_depot 2027330 2027331 3348281417896630206
What these numbers mean 2027330 = GoreBox App ID  ·  2027331 = GoreBox Depot ID  ·  last number = Manifest ID for your chosen version

Chapter 03

Coding the Plugin

Now the actual work begins. You'll write a BepInEx plugin that replaces GoreBox's default Photon and PlayFab credentials with your own.

1.
Install BepInEx on the downloaded version

Drop the BepInEx files into your GoreBox folder, then launch the game once. Wait for BepInEx to generate its interop files — this takes a minute. Close the game after it finishes loading.

2.
Create a new project in Visual Studio

Create a Class Library project targeting .NET 4.7.2. Name it whatever you want. Set the build configuration to Release (not Debug) from the start.

3.
Find VersionValidator in dnSpy

Open Assembly-CSharp.dll in dnSpy (located in your GoreBox GoreBox_Data/Managed folder). Search for VersionValidator. Note which lifecycle hook it uses — it varies by version:

Hook names vary by version Older versions may use Start or Awake; newer ones often use OnEnable or OnDisable. Check dnSpy for your specific version before writing the patch.

Here's what's inside VersionValidator and why it matters:

LoginResult — handles successful PlayFab logins. You'll hook this to redirect authentication to your own server. PlayFabError — handles login failures and displays ban messages. Both of these are wired differently in each GoreBox version, so always verify them in dnSpy.

Below is the full plugin code. Replace "YOUR_METHOD_NAME_HERE" with the actual method name you found in dnSpy, and fill in your Photon App IDs and PlayFab Title ID.

C# — Main Plugin 
using BepInEx;
using BepInEx.Unity.IL2CPP;
using HarmonyLib;
using Photon.Pun;
using PlayFab;
using PlayFab.ClientModels;
using UnityEngine.Networking;

namespace GoreBoxMain
{
    [BepInPlugin("GoreBox", "GoreBoxMain", "1.0")]
    public class GoreBoxClass : BasePlugin
    {
        public override void Load()
        {
            Harmony harmony = new Harmony("Harmony");
            harmony.PatchAll();
        }

        [HarmonyPatch(typeof(VersionValidator), "YOUR_METHOD_NAME_HERE")]
        public static class VersionValPatch
        {
            private static bool Prefix(VersionValidator __instance)
            {
                // Replace Photon credentials with your own
                PhotonNetwork.PhotonServerSettings.AppSettings.AppIdRealtime = "YOUR_PHOTON_APP_ID";
                PhotonNetwork.PhotonServerSettings.AppSettings.AppIdVoice = "YOUR_PHOTON_VOICE_ID";

                // Replace PlayFab credentials with your own
                PlayFabSettings.PlayFabSharedPrivate.TitleId = "YOUR_TITLE_ID";
                PlayFabSettings.PlayFabSharedPrivate.ProductionEnvironmentUrl = "https://YOUR_TITLE_ID.playfabapi.com";

                LoginWithCustomIDRequest loginRequest = new LoginWithCustomIDRequest
                {
                    CreateAccount = new Il2CppSystem.Nullable<bool>(true),
                    CustomId = UnityEngine.SystemInfo.deviceUniqueIdentifier
                };

                System.Action<LoginResult> onSuccess = delegate (LoginResult result)
                {
                    __instance.LoginResult(result); // Find the correct LoginResult in dnSpy
                };
                System.Action<PlayFabError> onError = delegate (PlayFabError err)
                {
                    __instance.PlayFabError(err); // Find the correct PlayFabError in dnSpy
                };

                PlayFabClientAPI.LoginWithCustomID(loginRequest, onSuccess, onError, null, null);

                return false; // Skip the original method
            }
        }
    }
}
⚠ LoginResult / PlayFabError names differ per version If multiplayer doesn't connect, look for other LoginResult or PlayFabError methods inside VersionValidator in dnSpy. They are renamed in almost every GoreBox update.

Chapter 04

Testing

1.
Build the DLL

In Visual Studio, set the configuration to Release and build the project. This produces a .dll file in the bin/Release folder of your project.

2.
Drop it into BepInEx

Copy your .dll into the BepInEx/plugins folder inside your GoreBox directory.

3.
Launch the game and watch the console

BepInEx opens a console window alongside the game. If a red error appears, something is wrong. Errors are highly specific — there is no universal fix list. Read the message carefully, check your method names in dnSpy, and verify your credentials.

4.
Test multiplayer

Try to join or create a multiplayer session. If you see a CustomAuthenticationFailed error, proceed to Chapter 5 — your PlayFab and Photon backends need to be configured.

CustomAuthenticationFailed? This is not a code bug — it means your PlayFab project isn't configured to allow the custom ID login method. Go to Chapter 5.
